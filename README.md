# 🤖 ADPozdnyakov_DemoBot

> 🚀 **Попробовать бота:** [@ADPozdnyakov_DemoBot](https://t.me/ADPozdnyakov_DemoBot)

Многофункциональный Telegram-бот, демонстрирующий широкий спектр возможностей Python-разработки: асинхронность, работа с API, кэширование, FSM (машина состояний), базы данных (SQLite), логирование, тестирование, CI/CD, контейнеризация (Docker) и парсинг веб-страниц и **обработку естественного языка (NLP)**.

## ✨ Возможности

| Категория | Команды |
|-----------|---------|
| **Погода** | `/weather` – текущая погода (OpenWeatherMap) |
| **Валюты** | `/currency` – конвертация валют с **распознаванием естественного языка** по курсу ЦБ РФ (с кэшированием 1 час) |
| **Аниме** | `/anime` – распознавание аниме по скриншоту (trace.moe) |
| **Перевод** | `/translate` – перевод текста на русский (Google Translate) |
| **Ссылки** | `/shorten` – сокращение ссылок (is.gd) |
| **QR‑коды** | `/qr` – генерация QR-кода из текста/ссылки |
| **Игры** | `/dice`, `/coin`, `/guess` – кубик, монетка, угадай число |
| **Животные** | `/cat`, `/dog` – случайные картинки котиков и собачек |
| **Контент** | `/joke`, `/fact` – случайные шутки и факты, `/quote` – цитаты |
| **Статистика** | `/stats` – общее число пользователей и команд, топ-5 команд |
| **Информация** | `/about` – технологии, версия, ссылка на GitHub |
| **Управление** | `/cancel` – отмена активного диалога, `/help` – список команд |

Все команды, требующие ввода данных, работают через **машину состояний (FSM)**: бот запрашивает нужную информацию последовательно. Во время диалога клавиатура заменяется на кнопку `/cancel`.

## 💱 Конвертер валют с NLP

Команда `/currency` поддерживает **три способа ввода** — выбирайте любой удобный!

### Способ 1: Всё в одном сообщении (быстрый)
Отправьте запрос одной фразой сразу после команды:

```
/currency 100 долларов в евро
/currency 13 фунтов в йенах
/currency 37 тенге в долларах
/currency 18 гонконгских долларов в канадские доллары
```

### Способ 2: Два сообщения (если забыли команду)
Можно сначала отправить `/currency`, а следующим сообщением — сам запрос:

```
Вы: /currency
Бот: Введите сумму...
Вы: 37 тенге в долларах
Бот: 37.0 KZT = 0.11 USD
```

### Способ 3: Пошаговый режим (FSM) — максимальная гибкость
Если хотите вводить данные по отдельности, просто следуйте подсказкам бота:

```
Вы: /currency
Бот: Введите сумму...
Вы: 37
Бот: Введите исходную валюту...
Вы: тенге          ← можно писать русскими буквами!
Бот: Введите целевую валюту...
Вы: доллар         ← или USD (оба варианта работают)
Бот: 37.0 KZT = 0.08 USD
```

**Важно:** в пошаговом режиме можно вводить валюты **как кодами** (`USD`, `EUR`, `RUB`), 
так и **русскими названиями** (`доллар`, `евро`, `рубль`) — бот поймёт и то, и другое.

**Что умеет распознавать:**
- ✅ Названия валют на русском (доллар, евро, фунт, йена, тенге, бат и др.)
- ✅ Падежи и формы слов (рублях, долларов, йенах → лемматизация)
- ✅ Составные названия (доллар сша, тайских бат, японских иен)
- ✅ Синонимы (йена → иена)
- ✅ Коды валют (USD, EUR, RUB) — работают в любом режиме

**Технологии под капотом:**
- **Natasha** – лемматизация и NER (распознавание именованных сущностей) для русского языка
- **Fallback-парсер** – если Natasha не сработала, используется умный поиск по словарю
- **Приоритет валют** – "доллар" по умолчанию = USD (США), а не AUD (по алфавиту первее)
- **Кэширование курсов** ЦБ РФ на 1 час

## 🛠 Технологии

- **Python 3.11+**
- **aiogram 3** – асинхронный фреймворк для Telegram Bot API
- **aiohttp** – асинхронные HTTP-запросы
- **SQLite** – хранение статистики пользователей и команд
- **Flask** – healthcheck для деплоя на Render
- **pytest** – unit-тесты
- **flake8, black** – линтеры и форматирование кода
- **GitHub Actions** – CI/CD (автоматические проверки и тесты)
- **Docker** – контейнеризация (Dockerfile + docker-compose)
- **Natasha** – библиотека для обработки русского языка (лемматизация, NER)
- **Логирование** – вывод в консоль и файл с ротацией (10 МБ, 5 бэкапов)

## 📦 Установка и запуск

### Локальный запуск (без Docker)

1. **Клонируйте репозиторий**
   ```bash
   git clone https://github.com/ShurikPozd/ADPozdnyakov_DemoBot.git
   cd ADPozdnyakov_DemoBot
   ```

2. **Создайте виртуальное окружение и активируйте его**
   ```bash
   python -m venv .venv
   source .venv/bin/activate      # Linux/macOS
   .venv\Scripts\activate          # Windows
   ```

3. **Установите зависимости**
   ```bash
   pip install -r requirements.txt
   ```

4. **Создайте файл `.env`** (скопируйте из `.env.example` и укажите свои токены)
   ```env
   BOT_TOKEN=ваш_токен_от_BotFather
   OWM_API_KEY=ваш_ключ_OpenWeatherMap
   ```

5. **Запустите бота**
   ```bash
   python bot.py
   ```

### 🐳 Запуск через Docker

1. **Убедитесь, что Docker установлен**.
2. **Соберите образ**
   ```bash
   docker build -t tg_demo_bot .
   ```
3. **Запустите контейнер**
   ```bash
   docker run --env-file .env -p 10000:10000 tg_demo_bot
   ```
   Или через `docker-compose`:
   ```bash
   docker-compose up --build
   ```

### ☁️ Деплой на Render

1. Загрузите код на GitHub.
2. На [Render.com](https://render.com) создайте **Web Service**.
3. Укажите репозиторий, build command: `pip install -r requirements.txt`, start command: `python bot.py`.
4. Добавьте переменные окружения `BOT_TOKEN` и `OWM_API_KEY`.
5. Сервис получит порт от Render, Flask healthcheck будет доступен по адресу `https://ваш-сервис.onrender.com/health`.

**Чтобы бот не засыпал на бесплатном тарифе**, рекомендуется настроить внешний мониторинг (например, UptimeRobot), который будет пинговать healthcheck-адрес каждые 5 минут.

## 🧪 Тестирование и линтеры

Unit-тесты написаны с использованием `pytest`:
```bash
pip install -r requirements-dev.txt
pytest -v
```

Тесты проверяют:
- Конвертацию валют (`convert_currency`)
- Форматирование времени (`format_time`)
- Загрузку случайных цитат (`get_random_quote`)

**Линтеры** (flake8, black) используются для поддержания чистоты кода. Проверить код можно командами:
```bash
flake8 . --count --statistics
black . --check
```

В CI/CD (GitHub Actions) линтеры и тесты запускаются автоматически при каждом пуше в ветку `main`.

## 📊 Статистика

Бот ведёт учёт пользователей и всех выполненных команд в SQLite. Команда `/stats` показывает:
- общее количество пользователей;
- общее количество команд;
- топ-5 самых популярных команд.

## 📜 Парсер цитат

В корне проекта находится скрипт `scripts/scrape_quotes.py`, который собирает цитаты с сайта `quotes.toscrape.com` (пагинация, авторы, теги) и сохраняет их в `data/quotes.json`. Этот JSON-файл используется командой `/quote` для выдачи случайных цитат. Скрипт демонстрирует навыки веб-парсинга (requests, BeautifulSoup) и работы с файлами.

Запустить парсер можно отдельно:
```bash
python scripts/scrape_quotes.py
```

## 🔍 Мониторинг доступности

Для предотвращения "засыпания" бота на бесплатном тарифном плане Render используется **Flask healthcheck** (эндпоинты `/` и `/health`). Рекомендуется настроить внешний мониторинг, например, **UptimeRobot** (бесплатно до 50 мониторов), который будет пинговать `https://ваш-сервис.onrender.com/health` каждые 5 минут. Это гарантирует, что сервис останется активным.
